      Web相关概念回顾

      1. 软件架构

      1.C/S：客户端/服务器端
2.B/S：浏览器/服务器端

      2. 资源分类

      1.静态资源：所用访问后，得到的结果都一样，可以直接被浏览器解析
    * html,css,JavaScript
2.动态资源：每个用户访问后，得到的结果可能不一样。动态资源被访问后，需要先转换为静态资源，再返回给浏览器
    * servlet/jsp,php,asp...

      3. 网络通信三要素

      1.IP：计算机在网络中的唯一标识
2.端口：应用程序在计算机中的唯一标识，0~65536.
3.通信协议：规定了数据通信的规则
    * TCP：安全的协议，三次握手，四次挥手。速度稍慢
    * UDP：不安全，速度快

      web 服务器软件

      * 服务器：安装了服务器软件的计算机
* 服务器软件：接收用户的请求，处理请求，做出响应
* web服务器软件：接受用户请求，处理请求，做出相应
    * 在web服务器软件中，可以部署web项目，用户可以通过浏览器访问这些项目
    * web容器
* 常见的Java 相关的web服务器软件：
    * weblogic：oracle公司，大型的JavaEE服务器，支持所有的JavaEE规范，收费的
    * webSphere：IBM公司，大型的JavaEE服务器，支持所有的JavaEE规范，收费的
    * JBOSS：JBOSS公司，大型的JavaEE服务器，支持所有的JavaEE规范，收费的
    * Tomcat：Apache基金会的，开源免费，中小型的JavaEE服务器，仅支持少量的JavaEE规范（servlet/jsp）   
* JavaEE：Java语言在企业级应用的技术规范的总和，一共规范了13项大的规范

      Tomcat

      1.Tomcat 目录文件

      2.Tomcat:下载，安装，卸载，启动，关闭，配置


      * 修改自身端口号。HTTP默认端口号是80.如果Tomcat使用80端口，则访问的时候端口号可省略

      HTTP协议：Hyper Text Transfer Protocol

      * 概念：定义了，客户端与服务器通信时的发送数据的格式。
* 特点：
    1.基于TCP/IP协议
    2.默认端口号：80
    3.基于请求/响应模型
    4.无状态协议：每次请求之间相互独立
* 1.请求消息：客户端发送给服务端的数据
    * 数据格式：
        1.请求行
        2.请求头
        3.请求空行
        4.请求体
* 2.响应消息：服务端发送给客户端的数据
    * 数据格式：
        1.响应行：
            格式：协议/版本 响应的状态码 状态码描述
        2.响应头：
            格式：头名称：值
            常见的响应头：
                1.content-type：服务端告诉客户端本次响应的数据格式以及编码格式
                2.content-disposition：服务器告诉客户端以什么格式打开响应数据，
                    *值：
                        * in-line：默认值，在当前页面打开
                        * attachment，filename-xxx：以附件的形式打开响应体，文件下载
        3.响应空行
        4.响应体：传输的数据

      会话技术:

      * 一次会话，包含多次响应与请求。从客户端第一次给服务端发送数据开始，到有一方断开为止。
* 建立会话的目的主要为了在一次会话范围内的多次请求间，共享数据。
* 建立会话的方式：
    1. 客户端会话技术：Cookie
    2. 服务端会话技术：Session

      1.Cookie:将数据保存在客户端的会话技术

      2.Session：将数据保存在服务端的会话技术

      软件三层架构：界面（web/表示层）、业务逻辑（service）、数据访问层（dao）

      Filter：过滤器

      * 作用：拦截请求，完成某些操作

      Listener：监听器

      * 事件监听机制
    * 事件
    * 事件源：事件发生的位置
    * 监听器：一个对象
    * 注册监听：将事件、事件源、监听器绑定到一起。当事件源上发生某事件时，执行监听器。

      AJAX：

      * 概念：异步JavaScript与XML（asynchronous JavaScript and XML）
